    # VuePress + Github Pages 搭建博客

    # 介绍

    一个极简的静态网站生成器。

    由Vue、Vue Router、webpack驱动的单页面应用。风格简约，配置简单。

    # 范例 && 博客推荐

    https://example.vuepress-theme-blog.billyyyyy3320.com/

    介绍 | Vue.js (vuejs.org) (opens new window)

    介绍 | vue-element-admin (panjiachen.github.io) (opens new window)

    awesome-bookmarks (panjiachen.github.io) (opens new window)

    三元博客 (sanyuan0704.top) (opens new window)

    # 0.前言

    以整合明天医网前端各类资源文档为例，记录简要流程示例

    # 1.本地搭建

    1.创建并进入一个新目录

    mkdir tomtaw-frontend && cd tomtaw-frontend

    2.使用包管理器进行初始化

    npm init

    3.将 VuePress 安装为本地依赖

    npm install -D vuepress

    4.创建你的第一篇文档

    mkdir docs
echo # Hello VuePress > docs/README.md

    5.在 package.json 中添加一些 scripts (opens new window)

    {
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}

    6.在本地启动服务器

    npm run docs:dev

    # 2.基础配置

    在docs目录下创建一个 .vuepress 目录，所有 VuePress 相关的文件都将会被放在这里。你的项目结构可能是这样：

    .
├─ docs
│  ├─ README.md
│  └─ .vuepress
│     └─ config.js
└─ package.json

    在 .vuepress 文件夹下添加 config.js，配置网站的标题和描述：

    module.exports = {
  title: 'Tomtaw Frontend',
  description: '明天医网前端'
}

    # 3.添加导航栏

    在config.js中添加以下配置：

    module.exports = {
	...
	themeConfig: {
        nav: [
          { text: '首页', link: '/' },
          { text: '开发指南', link: '/guide/'},
          { text: '技术分享', link: '/share/'},
          { text: '推荐库', link: '/repository/'},
        ]
    }	
}

    在docs目录下创建相应的目录及README.md。你的项目结构可能是这样：

    .
├─ docs
│  ├─ README.md
│  └─ .vuepress
│     └─ config.js
│  └─ guide
│     └─  README.md
│  └─ share
│     └─  README.md
│  └─ repository
│     └─  README.md
└─ package.json

    # 4.添加侧边栏

    现在我们添加一些 md 文档，文档的目录如下：

    .
├─ docs
│  ├─ README.md
│  └─ .vuepress
│     └─ config.js
│  └─ guide
│     └─  README.md
│     └─  javascript.md
│  └─ share
│     └─  README.md
│     └─  Vue-插件介绍.md
│     └─  Axios取消重复请求.md
│  └─ repository
│     └─  README.md
└─ package.json

    我们在 config.js 配置如下：

    module.exports = {
    ...
    sidebar: {
      '/guide/': [
          {
          title: '开发指南',
          path: '/guide/',
          children: [
              { title: "开发指南", path: "/guide/" },
              { title: "GIT规范", path: "/guide/javascript" },
          ]
        }
      ],
      '/share/': [
        {
          title: '技术分享',
          path: '/share/',
          children: [
            '',
            'vue插件介绍',
            'Axios取消重复请求',
        ]
        }
      ],
      '/repository/': [
        ''
      ],
      '/': [
          {
            title: '首页',
            path: '/'
          }
      ],
    }
}

    # 5. 更换主题

    现在基本的目录和导航功能已经实现，但如果我还想要加载 loading、切换动画、模式切换（暗黑模式）、返回顶部、评论等功能呢，为了简化开发成本，我们可以直接使用主题，这里使用的主题是 vuepress-theme-reco (opens new window)：

    现在我们安装 vuepress-theme-reco：

    npm install vuepress-theme-reco --save-dev

    然后在 config.js 里引用该主题：

    module.exports = {
  // ...
  theme: 'reco'
  // ...
}

    # 6. 添加文章信息

    ---
author: xxx
date: '2022-01-09'
---

    # 7. 设置语言

    module.exports = {
  // ...
  locales: {
    '/': {
      lang: 'zh-CN'
    }
  },
  // ...
}

    # 8. 开启目录结构

    module.exports = {
  //...
  themeConfig: {
    subSidebar: 'auto'
  }
  //...
}

    # 9. 修改主题颜色

    创建一个 .vuepress/styles/palette.styl 文件，文件代码如下：

    $accentColor = #3178c6

    # 10. 自定义修改样式

    在.vuepress/styles下创建一个index.styl文件，这是一个 Stylus 文件，但你也可以使用正常的 CSS 语法。

    .page .page-title .title{
   display: none;
}

    # 11. 部署

    到这里我们的博客就算是正式的做好了，接下来我们部署到免费的 Github Pages 上。我们在 Github 上新建一个仓库，这里我取得仓库名为：vuepress-starter。

    对应，我们需要在 config.js 添加一个 base 路径配置：

    module.exports = {
  	// 路径名为 "/<REPO>/"
    base: '/vuepress-starter/',
  	//...
}

    然后我们在项目根目录下建立一个脚本文件：deploy.sh，注意修改一下对应的用户名和仓库名：

    #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io/<REPO>
git push -f https://github.com/605511287/vuepress-starter.git master:gh-pages

cd -

    执行 sh deploy.sh，就会开始构建，然后提交到远程仓库。

    像我最后生成的地址就是 https://605511287.github.io/vuepress-starter/

    至此，我们完成了 VuePress 和 Github Pages 的部署。
